# GetYourOffer — Roadmap до готового продукта (v1)

Этот документ описывает путь от текущего состояния до **feature-complete v1** —
рабочего продукта dev-качества, покрывающего весь core-флоу из `AGENTS.md`.
`AGENTS.md` остаётся единым источником правды по продукту и конвенциям; здесь —
только план реализации: что значит «готово», какие фазы и в каком порядке.

Scope зафиксирован: **feature-complete v1** (tailor + аккаунты + фронтенд-авторизация
+ трекер откликов + история резюме + **freemium-заглушка**). Реальный биллинг,
деплой и CI — вне v1 (см. раздел «Что НЕ входит в v1»).

---

## 1. Что такое «готовый проект» (Definition of Done, v1)

Продукт считается готовым, когда все 5 MUST-пунктов core-флоу из `AGENTS.md`
(строки 29–37) работают **end-to-end для залогиненного пользователя** на чистом
русском дашборде:

1. ✅ Загрузка PDF-резюме + вставка текста вакансии → подогнанное резюме + анализ
   соответствия (совпадения + честные пробелы) + черновик сопроводительного письма.
2. Сохранение отклика в персональный **трекер** со статусами: черновик /
   отправлено / ответ / отказ / оффер.
3. **Аккаунт** (логин) + **история** откликов и версий резюме.
4. **Freemium-заглушка**: 3 бесплатные подгонки, дальше стаб «оформить доступ»
   (без реальной оплаты).
5. Всё связано единым дашбордом с навигацией, весь UI-текст — на русском.

**Инженерные критерии готовности v1:**

- `make fmt vet test` зелёный; фронт (`tsc -b && vite build`) собирается.
- Auth-путь покрыт тестами (сейчас не покрыт).
- JWT-секрет читается из конфига, не захардкожен.
- Новые env-переменные добавлены в `.env.example` и README.
- Резюме/вакансии/письма нигде не логируются (приватность из `AGENTS.md`).

## 3. Фазы

Порядок — из «Приоритета работ» `AGENTS.md`: сквозной срез уже готов; дальше
аккаунты → персистентность/история → трекер → freemium → полировка. Не начинаем
второстепенное, пока предыдущая фаза не работает целиком.

### Фаза 0 — Укрепление auth (enabling)

На идентичности пользователя держатся все следующие фазы, поэтому сначала делаем
её надёжной и тестируемой.

- JWT-секрет в конфиг: добавить `JWT_SECRET` в `config.go`, `.env.example`,
  README; заменить хардкод `temporary_secret_key` в
  `backend/internal/transport/http/user.go`; прокинуть через composition root.
- Валидация регистрации: формат email + непустой пароль (проверка в `service`,
  новый sentinel в `models/errors.go`).
- Проверка метода подписи (`*jwt.SigningMethodHMAC`) в keyfunc `AuthMiddleware`
  — гигиена против algorithm-confusion.
- Тесты auth-пути (TDD): `service/user`, хендлеры `user.go`, `AuthMiddleware`,
  `repository/user_postgres`.

**Файлы:** `backend/internal/config/config.go`,
`backend/internal/transport/http/user.go`, `backend/cmd/server/main.go`,
`.env.example`, `README.md`.

**Готово, когда:** секрет из env, регистрация валидируется, auth-путь под
тестами, `make test` зелёный.

### Фаза 1 — Фронтенд-авторизация

Активируем уже существующий бэкенд аккаунтов на фронте.

- Роутинг (`react-router-dom`) + app-shell + protected routes.
- Экраны Вход / Регистрация → `/api/users/register|login`; хранение JWT
  (localStorage); восстановление сессии через `GET /api/users/me`.
- Лёгкий auth-стейт (React Context + `useState`, без тяжёлых стейт-либ — в духе
  минималистичного фронта); в `api.ts` добавить `register/login/me` и заголовок
  `Authorization: Bearer …`.
- Logout, шапка с email пользователя.

**Файлы:** `frontend/src/App.tsx` (разбить на роуты/экраны),
`frontend/src/lib/api.ts`, новый `frontend/src/lib/auth.ts`, новые компоненты
страниц.

**Готово, когда:** пользователь регистрируется/входит/выходит; сессия переживает
перезагрузку; форма подгонки доступна под аккаунтом.

### Фаза 2 — Персистентность подгонок + версии резюме

Фундамент истории: каждая подгонка сохраняется и привязана к пользователю.

- Миграция `backend/migrations/0003_*.sql`: таблицы `resume_versions`
  (`user_id` FK, извлечённый текст, `created_at`) и `tailor_results`
  (`user_id` FK, `vacancy` text, `result` JSONB, `created_at`). Идемпотентно
  (`CREATE TABLE IF NOT EXISTS`).
- Привязать `/api/tailor` к пользователю (после Фазы 1 эндпоинт авторизованный),
  сохранять результат; расширить `service/tailor.go` + новый repository-адаптер
  (pgx, ошибки драйвера → sentinels).
- Эндпоинты: `GET /api/tailor/history`, `GET /api/tailor/history/{id}`.
- Фронт: экран истории + просмотр прошлого результата (переиспользовать
  `ResultPanel` из `App.tsx`).

**Файлы:** `backend/migrations/0003_*.sql`, `backend/internal/models/*`,
`backend/internal/service/tailor.go`, новый repository,
`backend/internal/transport/http/tailor.go`, `router.go`; экран истории на фронте.

**Готово, когда:** после подгонки результат виден в истории и открывается заново.

### Фаза 3 — Трекер откликов (core MUST)

- Миграция `backend/migrations/0004_*.sql`: таблица `applications`
  (`id`, `user_id` FK, `company`/`title`, `tailor_result_id` FK nullable,
  `status` enum `draft|sent|replied|rejected|offer`, `notes`, `created_at`,
  `updated_at`).
- Эндпоинты CRUD: create (часто из результата — «сохранить в трекер»), list,
  patch-статус/поля, delete. Новый `models/application.go` + repository +
  service + `transport/http/application.go`, монтаж в `router.go`.
- Фронт: экран трекера (список/доска по статусам), кнопка «Сохранить в трекер»
  из панели результата, смена статуса. Русские подписи статусов: черновик /
  отправлено / ответ / отказ / оффер.

**Файлы:** `backend/migrations/0004_*.sql`,
`backend/internal/models/application.go`, repository, service,
`backend/internal/transport/http/application.go`, `router.go`; экран трекера.

**Готово, когда:** отклик сохраняется в трекер и меняет статус среди 5 значений.

### Фаза 4 — Freemium-счётчик (заглушка)

- Учёт использований на аккаунт (счёт `tailor_results` за пользователя либо
  отдельный счётчик). Проверка лимита 3 в `service/tailor.go` → ответ-конверт
  «лимит исчерпан» (402/403) через новый sentinel в `models/errors.go`.
- Фронт: показ остатка бесплатных подгонок; при исчерпании — стаб «оформить
  доступ» (без оплаты, просто модалка/сообщение).

**Файлы:** `backend/internal/service/tailor.go`,
`backend/internal/transport/http/tailor.go`, `backend/internal/models/errors.go`;
UI-бейдж и стаб-модалка на фронте.

**Готово, когда:** после 3 подгонок появляется стаб «оформить доступ», подгонка
блокируется.

### Фаза 5 — Полировка дашборда и консистентность

- App-shell / навигация между экранами tailor / история / трекер; empty- и
  error-состояния; loading; адаптивность; единые Card и типографика.
- Обновить раздел «Функциональность» в README.
- Финальная проверка: `make fmt vet test` зелёный, фронт собирается.

**Файлы:** фронт по всем экранам; `README.md`.

**Готово, когда:** единый связный дашборд на русском, все критерии Definition of
Done выполнены.

## 4. Что НЕ входит в v1

Явно вне scope (из `AGENTS.md` + договорённости):

- Реальные платежи/биллинг (freemium — только заглушка-счётчик).
- Парсинг/скрапинг сайтов вакансий и job-бордов (вакансия вставляется текстом).
- Автоподача заявок за пользователя, интеграции с реальными ATS.
- Тренажёр собеседований (в роадмап, не в v1).
- Браузерное расширение.
- Email-верификация / сброс пароля.
- CI/CD, продакшн-хостинг сверх `docker-compose`, метрики/трейсинг/error-tracking.

## 5. Сквозные конвенции (соблюдать во всех фазах)

Из `AGENTS.md`:

- Коммиты на английском, императивный subject (~50–72 символа), **без
  AI-атрибуции** («Generated with…», «Co-Authored-By…» — нельзя).
- Пуш прямо в `main`, без PR и CI. Перед пушем: `make fmt vet test` и убедиться,
  что фронт собирается.
- Новые env-переменные сразу в `.env.example` и README.
- Layer-first архитектура; интерфейсы маленькие (1–3 метода) и на стороне
  потребителя; ошибки оборачиваем `%w`, не глотаем.
- Миграции — нумерованные идемпотентные SQL, применяются по порядку имён через
  `make db-migrate`.
- **Никогда не логировать** содержимое резюме/вакансий/писем — в логах только
  метод/путь/статус.
- Весь UI-текст — на русском; код, идентификаторы, комментарии — на английском;
  LLM генерирует на языке вакансии.

- [ ] Единый дашборд с навигацией по всем экранам, весь UI на русском.
- [ ] `make fmt vet test` зелёный; фронт собирается; auth-путь покрыт тестами;
